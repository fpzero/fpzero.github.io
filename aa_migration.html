The Basics

Page View Rules and Load Order

Adobe Analytics pageview requests are triggered by a Direct Call dlReady sent by GTM on this tag and based on some dataLayer events.

There are 8 rules on Adobe Launch that listen this Direct Call:

  • One general rule that runs on every dlReady direct call and set all global variables and send the Adobe Analytics pageview beacon
    • Global | AA - All Pages: dlReady [GTM]
  • Seven specific page rules that only runs on specific pages and set variables related to the page. These rules don’t send the Adobe Analytics pageview beacon.
    • Global | AA - Search Page: dlReady [GTM]
    • Global | AA - Product Listing Page: dlReady [GTM]
    • Global | AA - Product Detail Page: dlReady [GTM]
    • Global | AA - Shopping Cart page: dlReady [GTM]
    • Global | AA - Checkout pages: dlReady [GTM]
    • Global | AA - Order confirmation: dlReady [GTM]
    • Global | AA - Error Pages: dlReady [GTM]

Every rule has a Order number that determines which rule runs first when an event shared between rules is triggered. Rules with lower numbers runs first.

For the Adobe Analytics pageview rules, all the Specific pages rules has a smaller number than All Pages rules, forcing them to runs before the All Pages rule.

When Adobe Launch receives the dlReady Direct Call this will happen:

  1. All rules with the dlReady event will have its conditions checked;
  2. If the site page meets the condition of any of the 7 Specific page rules, this rule will run and set the related Adobe variables, without sending Adobe request;
  3. After this or if the page didn’t meet any rules condition, the All Pages rule will run. It will set global variables (e.g. pageName) and send the Adobe request;

In this way, if the user is in the Search page, the dlReady direct call will trigger first the Global | AA - Search Page: dlReady [GTM] and then it will trigger the Global | AA - All Pages: dlReady [GTM].
